# renpy-localize

A lightweight, offline Ren'Py translation tool powered by [Ollama](https://ollama.com).
Translates untranslated `.rpy` files using a local LLM — no API key, no cloud, no cost.

## Features

- **Fully offline** — runs entirely on your machine via Ollama
- **Zero dependencies** — pure Python standard library, no `pip install` needed
- **Tag protection** — Ren'Py tags (`{...}`, `[...]`) are masked during translation and restored afterwards, with automatic validation and retry
- **Context-aware** — passes preceding dialogue lines to the LLM for consistent translations
- **Batch translation** — translates multiple entries per API call for efficiency
- **Interrupt-safe** — saves partial progress on Ctrl+C or connection error
- **Auto backup** — creates timestamped backups before modifying any file
- **Force re-translate** — re-translate already translated entries with `--force`
- **Dry run** — preview translations without writing to files

## Requirements

- Python 3.10+
- [Ollama](https://ollama.com) running locally
- [Ren'Py SDK](https://www.renpy.org/latest.html) (to generate translation templates)

## Prerequisites: Generating Translation Templates

Before using this tool, you need to generate the translation template files with the Ren'Py SDK.

1. Open your game project in the Ren'Py launcher.
2. Go to **Generate Translations** (or run the following command):
   ```bash
   renpy.sh /path/to/your/game translate <language_name>
   ```
   Replace `<language_name>` with your desired locale identifier (e.g. `chinese`, `schinese`, `tchinese`).

3. Ren'Py will create a `tl/<language_name>/` folder inside your game directory. It mirrors the structure of your `game/` scripts:
   ```
   game/
   └── tl/
       └── chinese/
           ├── script.rpy       ← translation entries for script.rpy
           ├── chapter1.rpy     ← translation entries for chapter1.rpy
           └── common.rpy       ← translation entries for common.rpy
   ```

4. Each `.rpy` file in `tl/` contains empty translation entries like:
   ```renpy
   translate chinese start_12345678:
       # "Hello, world!"
       ""
   ```
   The comment line holds the original English text; the empty string `""` is where the translation goes.

Point `translate.py` at any of these files (or the whole `tl/chinese/` directory) and it will fill them in automatically.

## Setup

1. Install and start Ollama:
   ```bash
   ollama serve
   ollama pull qwen3:14b
   ```

2. Clone this repo:
   ```bash
   git clone https://github.com/ByLightLeak/renpy-localize.git
   cd renpy-localize
   ```

3. (Optional) Edit `config.json` to change the model or settings.

## Usage

```bash
# Translate a single file
python translate.py path/to/translation.rpy

# Translate all .rpy files in a directory
python translate.py path/to/tl/chinese/

# Recursively search subdirectories
python translate.py path/to/game/ -r

# Preview without writing
python translate.py path/to/translation.rpy --dry-run

# Re-translate already translated entries
python translate.py path/to/translation.rpy --force

# Use a different model
python translate.py path/to/translation.rpy --model qwen3:8b
```

## Configuration

Edit `config.json` to set your defaults:

```json
{
  "ollama_host": "http://localhost:11434",
  "model": "qwen3:14b",
  "batch_size": 5,
  "context_lines": 5,
  "num_ctx": 8192,
  "target_language": "Traditional Chinese (繁體中文)"
}
```

| Key | Description |
|-----|-------------|
| `ollama_host` | Ollama API URL |
| `model` | Ollama model name |
| `batch_size` | Entries per API call (1–20) |
| `context_lines` | Preceding dialogue lines passed as context |
| `num_ctx` | Ollama context window size (tokens) |
| `target_language` | Target language for translation |

You can also override `target_language` per-run with `--language`:

```bash
python translate.py path/to/translation.rpy --language "Japanese"
```

## Notes

- Default target language is **Traditional Chinese (繁體中文)**; set `target_language` in `config.json` or use `--language` to change it
- Works with **any model available in Ollama** — just set `model` in `config.json` or pass `--model <name>`. Any model with decent multilingual capability should work. `qwen3:14b` is the tested default, but lighter options like `qwen3:8b` or `qwen3:4b` are viable on machines with less VRAM.
- Designed for Ren'Py translation files generated by `renpy translate`

## 中文說明

輕量的離線 Ren'Py 翻譯工具，透過 [Ollama](https://ollama.com) 在本地執行 LLM 進行翻譯。
不需要 API 金鑰、不上雲端、完全免費。

### 特色

- **完全離線** — 透過 Ollama 在本機執行，不需網路
- **零依賴** — 只需 Python 標準函式庫，不需 `pip install`
- **Tag 保護** — 翻譯前自動遮蔽 Ren'Py tag（`{...}`、`[...]`），翻譯後還原並驗證，失敗自動重試
- **上下文感知** — 將前幾行對話傳給 LLM，確保翻譯前後一致
- **批次翻譯** — 每次 API 呼叫翻譯多筆，提升效率
- **中斷保護** — Ctrl+C 或連線失敗時自動儲存已完成的進度
- **自動備份** — 修改檔案前建立含時間戳的備份
- **強制重譯** — 用 `--force` 重新翻譯已有譯文的條目
- **預覽模式** — 用 `--dry-run` 預覽翻譯結果，不寫入檔案

### 需求

- Python 3.10+
- 本地執行中的 [Ollama](https://ollama.com)
- [Ren'Py SDK](https://www.renpy.org/latest.html)（用於產生翻譯模板）

### 前置步驟：產生翻譯模板

在使用本工具之前，你必須先用 Ren'Py SDK 產生翻譯模板檔案。

1. 在 Ren'Py 啟動器中開啟你的遊戲專案。
2. 點選 **Generate Translations**，或在命令列執行：
   ```bash
   renpy.sh /path/to/your/game translate <language_name>
   ```
   將 `<language_name>` 替換為你想要的語言識別碼（例如 `chinese`、`schinese`、`tchinese`）。

3. Ren'Py 會在遊戲目錄內建立 `tl/<language_name>/` 資料夾，結構與 `game/` 底下的腳本一致：
   ```
   game/
   └── tl/
       └── chinese/
           ├── script.rpy       ← script.rpy 的翻譯條目
           ├── chapter1.rpy     ← chapter1.rpy 的翻譯條目
           └── common.rpy       ← common.rpy 的翻譯條目
   ```

4. `tl/` 裡的每個 `.rpy` 檔案都包含空白的翻譯條目，格式如下：
   ```renpy
   translate chinese start_12345678:
       # "Hello, world!"
       ""
   ```
   注解行是原文，空字串 `""` 就是翻譯要填入的位置。

把 `translate.py` 指向這些檔案（或整個 `tl/chinese/` 目錄），它就會自動填入翻譯。
